# サービス料金シュミレーター

ポラリスデイサービスセンター万領専用の、1か月の利用料金目安を計算するStreamlitアプリです。

## 起動方法

```bash
pip install -r requirements.txt
streamlit run app.py
```

## Web公開方法

Streamlit Community Cloudで公開できます。

1. このフォルダのファイルをGitHubリポジトリにアップロードします。
2. Streamlit Community Cloudで「New app」を選びます。
3. リポジトリ、ブランチ、メインファイル `app.py` を指定します。
4. Advanced settings または App settings のSecretsに以下を設定します。
5. Deployを押すとWeb上で利用できます。

```toml
APP_PASSWORD = "自分だけが知っているパスワード"
```

公開URLを開くと、最初にパスワード入力画面が表示されます。

公開に必要な主なファイルは以下です。

- `app.py`
- `requirements.txt`
- `service_codes.csv`
- `.streamlit/config.toml`

## 自分だけで使う設定

外出先から自分だけが使う場合は、以下の形がおすすめです。

- GitHubリポジトリはPrivateにする
- Streamlit Community CloudでDeployする
- StreamlitのSecretsに `APP_PASSWORD` を設定する
- URLとパスワードを他の人に共有しない

ローカルで試す場合は、`.streamlit/secrets.example.toml` を参考にして `.streamlit/secrets.toml` を作成し、以下のように書きます。

```toml
APP_PASSWORD = "自分だけが知っているパスワード"
```

`.streamlit/secrets.toml` は `.gitignore` に入れているため、GitHubにはアップロードしない前提です。

Renderなど他のWebサービスに置く場合も、起動コマンドは以下を指定します。

```bash
streamlit run app.py --server.port $PORT --server.address 0.0.0.0
```

## 必要ライブラリ

- streamlit
- pandas

## 計算前提

- 地域区分は大阪市の2級地です。
- 地域単価は10.72円で計算します。
- 負担割合は1割、2割、3割から選択できます。
- 利用頻度ごとの月概算回数は固定です。
  - 週1回: 4回/月
  - 週2回: 9回/月
  - 週3回: 13回/月
  - 週4回: 17回/月
  - 週5回: 22回/月
  - 週6回: 26回/月
- 計算方法は「週の利用パターンで計算する」または「月の利用回数で計算する」から選べます。
- 月の利用回数で計算する場合は、月1回〜月31回まで選択できます。
- カフェ代は250円/回です。
- 要支援側は `20260601masuta_0522.csv` のA6サービスコードを正として、区分ごとのサービスコード・名称・単位数を使います。

## 要介護の計算

地域密着型通所介護の78コードを想定し、時間区分は「3時間以上4時間未満」で計算します。

基本単位は以下で固定しています。

| 介護度 | 単位数 |
| --- | ---: |
| 要介護1 | 416単位/回 |
| 要介護2 | 478単位/回 |
| 要介護3 | 540単位/回 |
| 要介護4 | 600単位/回 |
| 要介護5 | 663単位/回 |

加算は画面上のチェックボックスで選択できます。介護職員等処遇改善加算Ⅲは、基本単位と各種加算の合計に10.5%を掛け、`round()`で丸めています。

主な加算は以下です。

| 加算 | 単位数 |
| --- | ---: |
| 個別機能訓練加算Ⅰ1 | 56単位/回 |
| 個別機能訓練加算Ⅱ | 20単位/月 |
| 科学的介護推進体制加算 | 40単位/月 |
| ADL維持等加算Ⅰ | 30単位/月 |
| ADL維持等加算Ⅱ | 60単位/月 |
| サービス提供体制強化加算Ⅲ | 6単位/回 |

## サービスコードCSVの更新方法

要支援A6側は、CSVからA6コード・名称・単位数を読み込み、基本サービスと加算の計算に使用しています。

1. 最新のサービスコードCSVを用意します。
2. `20260601masuta_0522.csv` という名前でアプリと同じフォルダに配置します。
3. `service_codes.csv` は確認用のCSVです。正式なマスタCSVは `20260601masuta_0522.csv` として配置してください。

CSVは `utf-8-sig`、`cp932`、`shift_jis`、`utf-8` の順に読み込みを試します。

## 注意事項

この金額は概算です。実際の請求額は端数処理、給付管理、利用実績、加算算定状況等により変動する場合があります。
